# team66-hackaton

## Категории

Письма раскладываются по таким папкам внутри SortedInbox:

- Important — письма от адресов из белого списка
- Incidents — сбои, аварии, что-то упало или недоступно
- Security — фишинг, взлом, утечки, подозрительная активность
- Questions — вопросы, просьбы, заявки
- Spam — реклама, розыгрыши, сомнительные предложения
- Noreply — автоуведомления с адресов вида noreply, отвечать не нужно
- Other — всё, что не подошло ни под одну категорию
- Unreadable — файлы, которые не удалось прочитать, и письма без отправителя

## Как определяется категория

Для каждой категории мы задали список слов-сигналов. 
Часть слов считаем важными (вес 2), остальные обычными (вес 1). 
Слова из темы письма считаются с коэффициентом 3, из тела — с
коэффициентом 1. Письмо уходит в ту категорию, которая набрала больше всего
баллов, но только если балл не ниже порога. Если ничего не набралось —
письмо идёт в Other.

Отдельно проверяем: если отправитель есть в белом списке, письмо сразу
попадает в Important, а если в адресе есть noreply — в Noreply. Также
если письмо превышает минимальный балл для попадания в спам или автоответчики, 
оно уйдёт туда без подсчёта баллов за остальные категории.

Что будет в нестандартных случаях:
- письмо ни на что не похоже → уходит в Other, ничего не теряется
- файл не открывается или в письме нет отправителя → кладётся в Unreadable.
  Программа не падает и идёт дальше

## Директория проекта

- src/ — код
  - main.py — запуск и меню
  - FileReader.py — чтение писем
  - Classifier.py — классификация и перенос по папкам
  - Cleaner.py — очистка сортированного ящика и корзины
  - Settings.py — настройки (путь к ящику, белый список)
- tests/ — тесты
  - classifier_test.py — тесты для Classifier.py
  - fileReader_test.py — тесты для FileReader.py
- SortedInbox/ — сюда раскладываются письма
- TestFolder/ — тестовые письма, чтобы проверить работу
- UserInfo/ — настройки пользователя (path_To_Inbox.txt, white_list.txt)
- Logs/ — лог работы
- Trash/ — корзина
- run.sh, Starter.command, Starter.bat — запуск под Linux/mac и Windows

## Системные требования

Python 3.10 или новее. Для тестов нужен pytest (при запуске будет предложена установка)
